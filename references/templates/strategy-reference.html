Diagnosis

Design-to-code handoff is the last manual bottleneck in modern product teams

AI has collapsed the cost of writing code, but the translation from design intent to production implementation remains manual. Designers export specs, engineers interpret them, and the result is a constant stream of pixel-level mismatches, accessibility gaps, and wasted review cycles.

Figma Dev Mode and Storybook reduced friction but didn't eliminate it. Teams still spend 30-40% of frontend engineering time on design fidelity work — translating mockups into components that match intent. This is the gap Flux fills.

The window is narrow: Figma is investing in code generation, Vercel is building v0, and Cursor/Copilot are nibbling from the engineering side. We win by owning the space between design and merge — not design itself, not general coding, but the specific bridge.

Strategic bets

Three bets for the next 12 months

1
Design system ingestion as the moat
Import a team's design tokens, components, and patterns so generated code uses their system — not generic Tailwind. This makes output immediately mergeable and creates switching costs. No competitor does this today.
Not betting on: building our own design system. We ingest theirs.
2
AI design review as the expansion wedge
After code gen, Flux reviews the implementation against the design file and flags mismatches before the PR is opened. This replaces the designer's manual review cycle and expands our value beyond code generation into quality assurance.
Not betting on: replacing designers. We augment their review, not their creativity.
3
Figma-first, framework-agnostic
Go deep on Figma as the input format. Support React, Vue, and Svelte as output. Don't build a design tool. Don't build a code editor. Be the bridge that works with whatever tools teams already use.
Not betting on: Sketch, Adobe XD, or Penpot support. Figma is 80%+ of our ICP.

Guiding principles

Decision rails — when in doubt, these break the tie

01
Mergeable over impressive. A 90% fidelity component that uses the team's design system beats a 99% fidelity component with custom CSS that no one will maintain.
02
Bridge, don't replace. We connect Figma to code. We don't compete with Figma, VS Code, or GitHub. If a tool does something well, integrate with it.
03
Engineer trusts the output. Generated code should look like a senior engineer wrote it. If the engineer rewrites more than 10% of our output, we've failed.
04
Earn the workflow, don't demand it. No new tools to learn, no new tabs to open. Flux works inside the tools teams already use — Figma plugin, GitHub PR, CLI.
