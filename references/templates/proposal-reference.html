IProblem & Context

Non-English-speaking users bounce at onboarding because every label, tooltip, and help string is hardcoded in English.

28% of trial signups come from non-English markets but only 4% convert — a 7x gap vs. English users. The strategic driver is geographic expansion (Q2 priority). Competitors Acme and Globex already ship 5+ languages; we're losing deals to localized alternatives in LATAM and APAC.

Sales evidence — March 2026

"Three enterprise prospects in Japan asked for Japanese UI before signing POC agreements."

VFunctional Requirements

What each in-scope item must support, grouped by scope item.

1 · i18n framework

  • ICU MessageFormat for pluralization, gender, and number formatting.
  • Locale files in locales/{lang}.json, one file per supported language.
  • Build-time check fails if any locale is missing keys present in en.json.
  • Runtime fallback to English when a key is missing in the active locale.

2 · Language picker

  • Settings page section titled "Language" with three options.
  • Selection persists to user profile; applies instantly without reload.
  • Status indicator shows save state (saving / saved).

3 · String extraction

  • Replace all hardcoded English strings with t('key') calls.
  • Lint rule rejects new commits with raw English strings in JSX/templates.
  • Provide initial translations for Spanish and Japanese (professional translation vendor).

4 · Browser-locale detection

  • On first visit, read navigator.language and match against supported locales.
  • If matched, render in detected language; if not, default to English.
  • Show one-time banner: "We detected your browser is in {lang}. Continue or switch?"

VIEdge Cases & Constraints

Boundary conditions and failure modes — each must have a defined handling.

CaseExpected handling
Translation missing for active locale Fall back to English silently. Log to error tracker for translator backfill.
Browser locale is a region we don't ship Match by base language (e.g., es-MX → Spanish). Default to English if no base match.
String contains user variable (name, count) Use ICU MessageFormat placeholders — never concatenate translated fragments.
User switches language mid-task Re-render chrome immediately; preserve form state and unsaved input.
Translated string is significantly longer than English UI must reflow without truncation. CSS uses min-content for button labels.

IXTechnical Feasibility

Verdict: Feasible.

Build on: existing React component library, user profile schema, settings page infrastructure.

Build new: ICU MessageFormat integration, locale file structure, translation vendor pipeline, lint rule for hardcoded strings.

Top risks: string extraction touches ~340 files (mechanical but tedious — schedule a one-shot codemod). Translation vendor turnaround is 5–10 business days per language. Long-string reflow may surface latent CSS bugs.
