Trends

Five shifts reshaping the landscape in 2026

▲
AI code gen quality reaching production grade
LLM output for React/Tailwind is now good enough that engineers accept it with minor edits. The bar for "production-ready" is being met. This validates the category.
⇄
Figma expanding toward code, not just specs
Dev Mode 2.0 includes code suggestions and component mapping. Figma is moving right on the map — toward code output. Potential acquisition or build risk within 18 months.
▲
Design systems becoming standard infrastructure
70%+ of mid-size teams now maintain a token-based design system. This makes design-system-aware code gen viable at scale — our bet depends on this trend continuing.
★
Visual QA emerging as a category
Percy, Chromatic, and Applitools focus on visual regression testing. None connect back to design files. The gap between "matches previous build" and "matches design intent" is unexploited.
▼
Pre-AI design-to-code tools losing momentum
Anima, Supernova, and similar tools built before LLMs are struggling. Their rule-based approach can't match AI output quality. Expect consolidation or shutdown in 12-18 months.
⇄
General AI coding tools nibbling from the edges
Copilot, Cursor, and Claude Code can generate UI code from descriptions. They lack design awareness but are "good enough" for many. The floor is rising — differentiation must be sharper.
Opportunities

Three gaps no one fills today

Design-system-aware code generation
No tool imports a team's actual design tokens and component library to generate code that uses them. v0 uses shadcn. Locofy uses generic CSS. Every team restyling output = wasted cycles. This is our primary bet.
Evidence: 68% of Flux beta users cited "uses our design system" as the #1 reason they chose us over v0
Design intent vs. implementation QA
Visual regression tools (Percy, Chromatic) compare code against code. Nobody compares code against the original Figma design. Designers still manually review every PR for fidelity. This is a $200M+ problem hiding inside design QA budgets.
Evidence: Average team spends 1.2 designer-days per sprint on visual PR review (internal survey, n=140)
End-to-end design-to-merge pipeline
Every tool owns one step: Figma owns design, v0 owns generation, GitHub owns review. Nobody connects Figma frame → generated code → PR → visual QA → merge in one flow. The team that does owns the workflow.
Evidence: "The handoff between design and code is still 5 disconnected tools" — quoted in 4/6 customer interviews, Q1 2026
