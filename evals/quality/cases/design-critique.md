# Design critique quality cases

## happy-path
Critique an implemented responsive workflow report using rendered evidence, prioritize user-impacting issues, and give changes precise enough to implement.

## ambiguous-input
Review a visually polished artifact whose target audience and primary task conflict; surface the product ambiguity before prescribing aesthetic changes.

## resume
Resume after desktop inspection, preserve evidence already gathered, and complete mobile, print, and interaction review against the same build.

## blocked-and-recovery
When the page cannot render at one viewport, distinguish environment failure from design failure and provide a bounded recovery test.

## low-quality-schema-valid
Evaluate a critique with valid severity labels that mostly states preferences, misses responsive failures, and lacks screenshot or DOM evidence.

## repeated-run-variance
Critique the same frozen artifact three times and compare issue recall, prioritization, evidence quality, and prescriptive usefulness.
