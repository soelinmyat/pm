# PM Plugin Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a publishable Claude Code plugin that gives Product Managers structured workflows for product discovery, competitive intelligence, and feature grooming.

**Architecture:** Plugin repo with 5 skills (markdown), 6 commands (markdown), 1 agent (markdown), Node.js scripts for SEO API and visual companion server, HTML templates, and cross-platform manifests. All skills are platform-agnostic; platform differences handled by separate manifests and tool mapping docs.

**Tech Stack:** Markdown (skills), Node.js (zero-dep scripts), HTML/CSS/vanilla JS (visual companion + dashboard), JSON (config), Bash (hook scripts, server lifecycle)

**Spec:** `docs/superpowers/specs/2026-03-12-pm-plugin-design.md`

**Repo:** New standalone repo at `~/Projects/pm` (not inside cleanlog-mono)

Detect mode from arguments:

  (a) Update landscape overview, (b) Profile competitors, (c) Research a specific topic"
- anything else -> Topic Mode (treat argument as topic name)

## Landscape Mode (/pm:research landscape)

### When to Use
First research activity in a new project. Foundational artifact.

### Flow
1. If SEO provider configured: `node ${CLAUDE_PLUGIN_ROOT}/scripts/seo-provider.js getKeywords "{product domain}"`
2. Web search for market overview, key players, industry trends
3. Present findings for user validation
4. Write pm/landscape.md

### Landscape Document Structure
[Frontmatter: type: landscape, created, updated, sources]
Sections: Market Overview, Key Players (table), Keyword Landscape (from SEO data),
Market Segments, Initial Observations

### Update Flow
Re-run refreshes data and appends new findings. Track what changed.

## Competitor Mode (/pm:research competitors)

### Phase 1: Discover
- If pm/landscape.md exists: use Key Players table as candidate list
- If no landscape: use SEO + web search (recommend running landscape first)
- Present candidates to user for confirmation
- Save to pm/competitors/index.md

### Phase 2: Profile
- **2+ competitors, subagents available (Claude Code, Codex):** Dispatch one researcher agent
  per competitor (parallel). Each agent writes to its own `pm/competitors/{slug}/` directory.
  `Agent tool: name="researcher", prompt="Profile {name} in the {space} space. Slug: {slug}.
  Follow methodology in skills/research/competitor-profiling.md.
  Write to pm/competitors/{slug}/"`
- **2+ competitors, NO subagents (Gemini, OpenCode, Cursor):** Profile competitors
  sequentially inline, one at a time, following competitor-profiling.md methodology.
  Confirm with user between each competitor ("Finished profiling {name}. Continue to next?").
- **1 competitor (any platform):** Profile inline following competitor-profiling.md methodology.

**Subagent detection:** Try dispatching via the Agent tool. If the tool is unavailable or
returns an error, fall back to sequential inline profiling. Do not fail.

**Index ownership:** Researcher agents do NOT write to pm/competitors/index.md.
Only the parent research skill updates the index after all agents complete (Phase 3).

### Phase 3: Synthesize
- Update pm/competitors/index.md with all profiled competitors
- Generate comparison matrix across all profiled competitors
- Identify market gaps and opportunities
- Visual companion: interactive competitor comparison table

### Cost Guardrail
Before batch SEO operations: "About to run SEO analysis on {N} competitors.
This will use approximately {estimate} API calls. Proceed?"

## Topic Mode (/pm:research {topic})

1. Check existing: pm/research/{topic}/ and pm/competitors/ for relevant data
2. Check strategy: pm/strategy.md for alignment context
3. Web search + SEO for gaps
4. Write to pm/research/{topic-slug}/findings.md with frontmatter
5. Update pm/research/index.md (add entry: topic, status, last updated)

### Index Format (pm/research/index.md)
| Topic | Status | Last Updated |
|-------|--------|--------------|
| {topic} | complete | 2026-03-12 |

## SEO Provider Invocation
All SEO calls via Bash: `node ${CLAUDE_PLUGIN_ROOT}/scripts/seo-provider.js {command} {args}`
Commands: getKeywords, getTraffic, getBacklinks, getCompetitors
If provider is "none": skip SEO steps, proceed with web search only.
Handle errors: if script returns {"error": "..."}, show error and continue without SEO data.

- Always check existing knowledge before web search
- Save with sources and access dates
- Update existing files, don't create duplicates
- No artificial limit on search depth
