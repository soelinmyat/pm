Simple, Readable Output

Every message scannable in 5 seconds. Verdicts first. Bullets, not paragraphs. Plain language a 16-year-old gets at a glance.

Problem

Users skip reading plugin output because it's too dense.

The plugin creator reports saying "yes" to review gates without reading them — because scanning paragraphs of jargon-heavy analysis is exhausting. If the person making product decisions bypasses review gates, the gates aren't working.

Root cause: The skill prompts don't constrain output density. Scripted messages are short, but the AI's analysis paragraphs (scope reviews, team reviews, strategy checks) are uncontrolled walls of text.

Scope

Style guide + scannable proposal template. Visual companion deferred.

In Scope

  • Writing style guide (grade 8, plain language, before/after examples)
  • SKILL.md formatting rules (verdict first, max 3 bullets, no walls of text)
  • Proposal template rewrite (one-sentence summaries, collapsible ACs)
  • TL;DR-first pattern for review outputs
  • Max-density rule + parallel agent output collapsing
10x Differentiator

Out of Scope

  • Visual companion (PM-036, own initiative)
  • Dashboard server changes
  • Review agent prompt rewrites
  • Pipeline structure changes
  • Internationalization

Competitive Context

No competitor constrains AI output to a readability standard. PM would be first.

ChatPRD recognized this pain — users complained about verbosity, so they added a concise/balanced/detailed toggle (Dec 2025). That's a user-controlled bandaid. PM solves it structurally: formatting rules enforced at the prompt layer, not a toggle. Productboard Spark users cite "overwhelming interface" as a top complaint. PM Skills Marketplace has no readability controls at all.
