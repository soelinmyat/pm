## Outcome

The dashboard separates bugs/issues from feature proposals and ideas. The user can quickly see what's broken vs what's planned without scanning a mixed list.

## Open Questions

- Should "Issues" be a new top-level nav item (Home, Backlog, Issues, Research)?
- Or a tab within Backlog (Proposals | Issues | Shipped)?
- What qualifies as an "issue" vs a backlog item? Label-based (`bug` label) or status-based?
- Should bugs have a different card style (red accent vs green/blue)?
- How does this interact with the kanban view (Ready to Build, Ideas, Shipped)?

## Notes

- Currently bugs, features, and ideas all live in the same backlog list with the same card style.
- PM-094 and PM-095 are bugs that show as "idea" or "approved" — indistinguishable from features visually.
- Needs design exploration before implementation.
