Toast Notification Behavior
  • Appears bottom-right of main content area (not over the feed panel)
  • Auto-dismisses after 3-5 seconds (500ms per word + 1s buffer)
  • Subtle slide-in animation. Respects prefers-reduced-motion.
  • Only for milestone events: PR created, tests passed, review done, merged
  • No interactive elements — pure confirmation, not actionable
  • Queue multiple toasts without overlapping

Event Schema (POST /events body)
  • type (string, required): Event category — "tests_passed", "pr_created", "review_done", "phase_started", "pushed", "merged"
  • source (string, required): Terminal session identifier — e.g., "terminal-1", process PID, or session slug
  • timestamp (number, required): Epoch milliseconds
  • detail (object, optional): Event-specific data — e.g., { tests: 8, passed: 8 } or { pr: 47, branch: "feat/sse" }
  • source_type (string, optional): "terminal" for v1. Extensible for future "webhook" or "integration" sources.
